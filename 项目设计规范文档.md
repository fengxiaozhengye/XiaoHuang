# 基于大模型的个性化资源生成与学习多智能体系统

## 项目设计规范文档

---

## 一、项目概述

### 1.1 项目背景

在数字化与智能化深度融合的时代，高等教育的个性化变革成为核心发展方向。不同学生在知识基础、学习能力、兴趣方向上存在显著差异，标准化教学难以满足个性化学习需求。当前大模型技术高速发展，以通用大模型、多模态生成大模型、AI辅助编程工具为代表的技术体系，为高等教育领域的创新升级带来全新契机。

### 1.2 项目目标

本项目旨在借助大模型技术体系，构建一个面向高等教育的个性化学习资源生成与多智能体辅导系统，实现以下目标：

- **学生画像系统**：精准分析学生知识水平、学习风格、薄弱环节
- **个性化学习路径**：基于知识图谱自动生成适配学生能力的学习路径
- **多模态资源生成**：根据学生水平动态生成文本、代码示例、练习题等学习资源
- **智能辅导对话**：多智能体协作的实时答疑与苏格拉底式引导教学
- **评估反馈闭环**：持续评估学习效果，动态更新画像和路径
- **通用课程框架**：支持教师自主导入任意课程，系统自动构建知识图谱

### 1.3 关键技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| AI模型 | 通义千问 / DeepSeek | 国产大模型，通过OpenAI兼容协议对接Spring AI，成本低、延迟小 |
| 课程范围 | 通用框架 | 不限定具体课程，支持教师自主导入知识点和依赖关系 |
| 前端样式 | 纯Ant Design | 组件库自带完整设计体系，开发效率高，风格统一 |

### 3.2 多智能体架构

系统包含5个专职Agent，由Orchestrator统一编排调度：

#### Agent 1: 诊断Agent (DiagnosisAgent)

| 属性 | 说明 |
|------|------|
| 职责 | 分析学生知识水平，构建/更新学生画像 |
| 输入 | 初始测评问卷答案、历史成绩、学习记录 |
| 输出 | 学生画像JSON（知识掌握度、学习风格、薄弱点、优势点） |
| 工具 | `StudentQueryTool`（查询学生历史）、`KnowledgeGraphTool`（查询知识点体系） |
| 触发时机 | 学生首次注册完成测评、定期复测 |

**Prompt模板核心指令：**
```
你是一名教育诊断专家。根据学生的测评结果和学习记录，分析其：
1. 各知识点的掌握程度（0-1分值）
2. 学习风格偏好（视觉型/阅读型/练习型）
3. 薄弱环节列表
4. 优势领域列表
请以JSON格式输出结构化的学生画像。
```

#### Agent 2: 规划Agent (PlanningAgent)

| 属性 | 说明 |
|------|------|
| 职责 | 基于学生画像和知识图谱，生成个性化学习路径 |
| 输入 | 学生画像 + 课程知识图谱 + 学习目标 |
| 输出 | 有序知识点序列 + 每个知识点推荐的资源类型和难度 + 时间安排 |
| 工具 | `KnowledgeGraphTool`（查询知识点依赖）、`ResourceQueryTool`（查询已有资源） |
| 触发时机 | 学生选择课程开始学习、评估Agent建议路径调整时 |

**核心策略：**
- 跳过已掌握的知识点（掌握度 > 0.8）
- 优先安排薄弱环节的前置依赖
- 根据学习风格调整资源类型比例

#### Agent 3: 生成Agent (GenerationAgent)

| 属性 | 说明 |
|------|------|
| 职责 | 根据学习需求动态生成多模态学习资源 |
| 输入 | 知识点 + 学生水平 + 资源类型 + 难度要求 |
| 输出 | 文本讲解、代码示例、练习题、思维导图描述 |
| 工具 | LLM生成能力、`ResourceQueryTool`（避免重复生成） |
| 触发时机 | 学习路径推进到新知识点、学生请求额外资源 |

**难度自适应：**
- 初学者（掌握度 < 0.3）：基础概念 + 生活类比 + 简单示例
- 中级（0.3-0.7）：标准讲解 + 典型应用 + 中等难度练习
- 高级（> 0.7）：深入原理 + 边界情况 + 综合应用题

#### Agent 4: 辅导Agent (TutorAgent)

| 属性 | 说明 |
|------|------|
| 职责 | 实时答疑、苏格拉底式引导教学 |
| 输入 | 学生提问 + 当前学习上下文 + 对话历史 |
| 输出 | 引导性回答、知识点提示、反问引导、相关练习推荐 |
| 工具 | `RagRetrievalTool`（检索教材原文）、`StudentQueryTool`（查询学习状态） |
| 触发时机 | 学生在对话界面提问（实时，SSE流式响应） |

**教学策略：**
- 不直接给答案，通过反问引导学生思考
- 引用教材原文增加可信度（RAG）
- 根据学生水平调整解释深度

#### Agent 5: 评估Agent (EvaluationAgent)

| 属性 | 说明 |
|------|------|
| 职责 | 评估学习效果，更新学生画像，建议路径调整 |
| 输入 | 练习/测试结果、学习行为数据（时长、完成率） |
| 输出 | 知识点掌握度更新、薄弱点变化、路径调整建议 |
| 工具 | `StudentQueryTool`（更新画像） |
| 触发时机 | 学生完成练习/测试、学习行为数据积累到阈值 |

#### Agent间通信机制

```
同步场景（实时响应）：
  辅导Agent → 直接调用，SSE流式返回

串行编排（学习流程）：
  诊断Agent → 规划Agent → 生成Agent （通过Orchestrator串行调度）

异步触发（评估更新）：
  学习行为 → Redis Stream → 评估Agent消费 → 更新画像 → 触发路径调整
```

## 四、数据库设计

### 4.1 MySQL 业务数据库

#### 用户表 (user)

```sql
CREATE TABLE `user` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `username` VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    `password` VARCHAR(255) NOT NULL COMMENT '密码（BCrypt加密）',
    `real_name` VARCHAR(50) COMMENT '真实姓名',
    `email` VARCHAR(100) COMMENT '邮箱',
    `avatar_url` VARCHAR(500) COMMENT '头像地址',
    `role` ENUM('STUDENT', 'TEACHER', 'ADMIN') NOT NULL DEFAULT 'STUDENT' COMMENT '角色',
    `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态：1-正常 0-禁用',
    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX `idx_role` (`role`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

#### 学生画像表 (student_profile)

```sql
CREATE TABLE `student_profile` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,

    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX `idx_user` (`user_id`),
    INDEX `idx_status` (`status`),
    FOREIGN KEY (`user_id`) REFERENCES `user`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户上传文档表（个人知识库）';
```

### 4.2 PostgreSQL 向量数据库

```sql
-- 启用向量扩展
CREATE EXTENSION IF NOT EXISTS vector;

-- 文档向量嵌入表（系统课程教材 + 个人知识库）
CREATE TABLE document_embedding (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT COMMENT '用户ID（NULL=系统公共文档，非NULL=个人知识库）',
    course_id BIGINT COMMENT '关联课程ID',
    document_id BIGINT COMMENT '关联user_document表ID（个人文档）',
    knowledge_point_id BIGINT COMMENT '关联知识点（Agent标注或手动关联）',
    content TEXT NOT NULL COMMENT '原始文本块',
    metadata JSONB COMMENT '来源：{"source": "教材第3章"/"用户上传", "page": 42, "file_type": "PDF"}',
    embedding vector(1024) COMMENT '向量嵌入（通义千问text-embedding-v3维度1024）',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 向量相似度索引
CREATE INDEX idx_embedding ON document_embedding
    USING ivfflat (embedding vector_cosine_ops)
    WITH (lists = 100);

-- 按用户+课程过滤索引
CREATE INDEX idx_user_course ON document_embedding (user_id, course_id);
CREATE INDEX idx_document ON document_embedding (document_id);
```

```

```
用户上传文件 (PDF/Word/图片/TXT/MD)
       │
       ▼
┌──────────────────────────────────────────────┐
│ 1. 文件上传                                   │

└──────────────────┬───────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────┐
│ 2. 内容提取（异步，CompletableFuture）         │
│    - 更新状态: PARSING                        │
│    - PDF → PDFBox 提取文本+表格               │
│    - Word → Apache POI 提取段落               │
│    - 图片 → 通义千问VL多模态识别（OCR）        │

└──────────────────┬───────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────┐
│ 3. 文本分块                                   │

                   │
                   ▼
┌──────────────────────────────────────────────┐

│    - 更新状态: VECTORIZING                    │
│    - 调用 Embedding API（text-embedding-v3）  │

└──────────────────┬───────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────┐
│ 5. 完成                                       │
│    - 更新状态: COMPLETED                      │
│    - 更新 total_chunks 数量                   │

**内容提取方案详情：**

| 文件类型 | 解析方案 | Java库 | 说明 |
|---------|---------|--------|------|
| PDF | 文本提取 + 表格识别 | Apache PDFBox 3.0 | 支持文字PDF，扫描PDF需OCR |
| Word (.docx) | 段落解析 | Apache POI 5.x | 提取标题、正文、表格 |
| 图片 (jpg/png) | 多模态大模型OCR | 通义千问VL API | 能识别文字、图表、公式、手写笔记 |
| 纯文本 (.txt) | 直接读取 | Java IO | — |
| Markdown (.md) | 直接读取 | Java IO | 保留格式标记 |

**文本分块策略：**

```java
// 语义分块核心逻辑
public class TextChunker {

    private static final int MAX_CHUNK_SIZE = 512;  // tokens
    private static final int OVERLAP_SIZE = 64;     // tokens

    public List<TextChunk> chunk(String text, Map<String, Object> metadata) {
        // 1. 按段落分割（\n\n）
        List<String> paragraphs = splitByParagraph(text);

        // 2. 合并小段落直到接近 MAX_CHUNK_SIZE
        // 3. 拆分超大段落（按句子边界）
        // 4. 相邻块添加 OVERLAP 重叠
        // 5. 每块附加元数据：{source, page, chunk_index}

        return chunks;
    }
}
```
